# 4개 간행지 운영 설계

이 문서는 `Project_Newsletter`를 하나의 종합 뉴스레터가 아니라
독자별 4개 간행지로 운영하기 위한 실무 설계안입니다.

---

## 1. 운영 방향

### 권장 구조

- 하나의 브랜드 아래 4개 에디션 운영
- 공통 원칙은 유지하고, 독자와 톤만 분화
- 완전 독립 매체처럼 분리하기보다 "시리즈형 간행지"로 시작

### 권장 브랜드 예시

- `Project Newsletter Game`
- `Project Newsletter IT`
- `Project Newsletter Crypto`
- `Project Newsletter Football`

### 공통으로 유지할 것

- 쉬운 설명
- 팩트 체크 우선
- 출처 명시
- 짧고 읽기 쉬운 Markdown 포맷
- "왜 중요한지"를 설명하는 편집 방식

---

## 2. 독자 페르소나

각 간행지는 주제가 아니라 "누가 읽는지"를 먼저 기준으로 삼아야 합니다.

### A. 게임 간행지

**핵심 독자**

- 20대~30대 게이머
- 신작, 플랫폼 변화, 업계 이슈를 빠르게 알고 싶은 사람
- 게임 커뮤니티는 보지만 긴 기사까지는 잘 안 읽는 독자

**상황**

- 출퇴근 중 가볍게 읽음
- "이 게임이 왜 화제인지"를 빠르게 알고 싶어함
- 너무 업계식 표현보다는 체감 위주의 설명을 선호

**필요**

- 신작 발표 요약
- 플레이 경험에 어떤 변화가 생기는지 설명
- 과장 없는 기대 포인트와 리스크 구분

**싫어하는 것**

- 지나치게 기자 문체인 글
- 게임 커뮤니티 밈을 과하게 쓰는 글
- 근거 없는 흥행 예측

### B. IT 간행지

**핵심 독자**

- 25대~40대 직장인, 개발자, PM, 기획자
- AI, 플랫폼, 서비스 변화가 일과 어떻게 연결되는지 궁금한 사람
- 헤드라인은 보지만 기술 기사 원문은 다 읽지 않는 독자

**상황**

- 업무 전후로 훑어봄
- 기술 자체보다 "일에 어떤 영향이 있는지"를 궁금해함
- 과장된 미래 예측보다 실무 감각을 원함

**필요**

- 제품 발표의 실제 의미
- 업무 도구, 플랫폼, 정책 변화 정리
- 과열된 AI 홍보와 실제 효용의 구분

**싫어하는 것**

- 지나친 팬심
- 기술 용어 남발
- 검증 안 된 혁신 서사

### C. 코인 간행지

**핵심 독자**

- 20대~40대 투자 관심층
- 코인에 관심은 있지만 차트만으로 판단하고 싶지 않은 사람
- 뉴스가 가격과 제도, 리스크에 어떤 영향을 주는지 알고 싶은 독자

**상황**

- 짧은 시간에 핵심만 파악하려 함
- 숫자와 규제 이슈에 민감함
- 감정적 전망보다 구조 설명을 원함

**필요**

- 가격보다 이벤트의 의미 설명
- 거래소, 정책, ETF, 체인 이슈 해설
- "사실", "가능성", "전망"의 분리

**싫어하는 것**

- 투자 권유처럼 들리는 문장
- 확신형 전망
- 루머를 사실처럼 쓰는 방식

### D. 축구 간행지

**핵심 독자**

- 10대 후반~40대 축구 팬
- 경기 결과는 이미 알지만, 맥락과 해석을 보고 싶은 독자
- 이적, 전술, 구단 운영 이슈를 쉽게 이해하고 싶은 사람

**상황**

- 경기 하이라이트는 봤지만 긴 분석글은 부담스러움
- "왜 이 선수가 중요하지?"를 알고 싶어함
- 팬덤 감정은 있지만 너무 편향된 글은 싫어함

**필요**

- 경기 결과보다 의미 설명
- 이적의 실제 영향
- 전술 변화의 쉬운 번역

**싫어하는 것**

- 팬 커뮤니티식 과몰입 문장
- 확인 안 된 이적 루머 확대
- 결과만 나열하는 기사식 글

## 3. 간행지별 톤 차이

기본 베이스는 모두 "친근하고 쉽게"이지만, 말투의 무게중심은 달라야 합니다.

### 공통 톤

- 설명은 쉽게
- 말투는 부드럽게
- 팩트와 의견은 분리
- 독자를 가르치기보다 함께 정리하는 느낌

### 게임 간행지 톤

**키워드**

- 생동감
- 체감형 설명
- 기대와 우려를 균형 있게

**문장 느낌**

- "이 게임이 왜 눈길을 끄는지"
- "플레이어 입장에서 뭐가 달라지는지"
- "기대 포인트와 걱정 포인트를 같이 보기"

**피해야 할 것**

- 과한 텐션
- 유행어 남발
- "무조건 대박" 같은 흥행 단정

### IT 간행지 톤

**키워드**

- 차분함
- 실무적 해석
- 과장 제거

**문장 느낌**

- "이 발표가 실제로 의미하는 것"
- "사용자나 업계 입장에서 보면"
- "당장 바뀌는 것과 아직 먼 이야기를 구분"

**피해야 할 것**

- 광고 문구처럼 들리는 표현
- 기술 낙관론 일변도
- 전문 용어 설명 생략

### 코인 간행지 톤

**키워드**

- 절제
- 숫자와 리스크 중심
- 전망보다 구조 설명

**문장 느낌**

- "가격보다 먼저 봐야 할 건"
- "이건 사실이고, 그다음은 가능성"
- "변동성이 큰 만큼 단정은 피하기"

**피해야 할 것**

- 투자 조언처럼 읽히는 표현
- 공포/탐욕 자극 문장
- 루머를 확대하는 서술

### 축구 간행지 톤

**키워드**

- 현장감
- 맥락 설명
- 팬심은 이해하되 편향은 절제

**문장 느낌**

- "결과보다 내용이 중요했던 경기"
- "이 영입이 시즌 전체에 미치는 영향"
- "전술을 쉽게 말하면 이런 변화"

**피해야 할 것**

- 특정 팀 편들기
- 감정 과열
- 루머성 이적 기사 재포장

## 4. 추천 편집 포맷

### 공통 기사 단위 구조

1. 제목
2. 한 줄 요약
3. 무슨 일인지
4. 왜 중요한지
5. 독자가 알아둘 포인트
6. 출처

### 간행지별 비중 차이

- 게임: "왜 기대받는지"와 "유저 체감 변화" 비중 확대
- IT: "업무/산업 영향" 비중 확대
- 코인: "사실/수치/리스크" 비중 확대
- 축구: "맥락/전술/시즌 영향" 비중 확대

### 왜 이 구조가 좋은가

- 간행지별 소스, 초안, 발행본이 분리됨

- 자동화가 생겨도 간행지 단위로 붙이기 쉬움
- 향후 일부 간행지만 독립 운영해도 구조 변경이 적음

## 7. 운영 방식 제안

### 시작 단계

- 처음부터 4개를 모두 매주 발행하지 않기
- `IT`, `Game` 2개부터 시작하거나
- 4개를 운영하되 격주 발행으로 시작

### 추천 발행 방식

- Game: 주 1회
- IT: 주 1회
- Crypto: 주 1회 또는 시장 변동이 클 때만 추가판
- Football: 주 1회

### 최소 운영 인력 기준

- 1인 운영: 주당 2개 간행지가 현실적
- 2인 이상: 4개 병행 가능
- 자동화 도입 전까지는 "많이 발행"보다 "꾸준히 발행"이 더 중요

## 8. 실무 체크포인트

### 페르소나 기준 질문

- 이 뉴스는 이 간행지 독자가 정말 궁금해할 주제인가?
- 이 독자는 제목만 보고도 읽고 싶어질까?
- 너무 넓은 독자를 상정해 설명이 흐려지지 않았는가?

### 톤 기준 질문

- 게임인데 IT 기사처럼 딱딱하지 않은가?
- IT인데 홍보문처럼 들리지 않는가?
- 코인인데 전망 장사처럼 읽히지 않는가?
- 축구인데 팬 커뮤니티 감정에 끌려가지 않았는가?

### 구조 기준 질문

- 각 간행지 자료가 서로 섞이지 않는가?
- 출처와 초안과 발행본이 분리되어 있는가?
- 자동화가 들어와도 파일 위치가 명확한가?

## 9. 우선순위 제안

바로 전부 뜯어고치기보다 아래 순서가 안전합니다.

1. `docs/style-guides/`에 공통/개별 톤 가이드 작성
2. `templates/editions/`에 4개 전용 템플릿 생성
3. `editions/` 구조 생성
4. 기존 샘플 초안을 한 간행지 기준으로 옮겨보기
5. 이후 자동화 스크립트 분리

---

## 10. 결론

이 프로젝트는 4개 간행지로 분화하기에 충분한 기초를 갖고 있습니다.
다만 핵심은 "분야를 나눈다"가 아니라 "독자를 다르게 상정한다"입니다.

가장 현실적인 전략은 다음과 같습니다.

- 브랜드는 하나로 유지
- 간행지는 4개로 분화
- 공통 원칙은 유지
- 톤과 편집 기준은 독자에 맞게 조정
- 폴더 구조는 edition 중심으로 재편

이 방식이면 지금의 프로젝트 철학을 유지하면서도
운영성과 확장성을 함께 가져갈 수 있습니다.
